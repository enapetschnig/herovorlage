### packages/integrations/zugferd — XRechnung CII XML Generator
- Renders **EN 16931 / XRechnung 3.0 konformes** UN/CEFACT Cross-Industry-Invoice 100 BASIC XML
- Korrekte ZUGFeRD-Profile-URI in `GuidelineSpecifiedDocumentContextParameter`
- TypeCode-Mapping (84=Angebot, 380=Rechnung, 381=Gutschrift, 270=Lieferschein, 326/385 für Teil-/Schlussrechnung)
- VAT-Buckets pro Steuersatz korrekt aggregiert in `ApplicableTradeTax`
- Unit-Code-Mapping (Stk→H87, m→MTR, h→HUR, etc., Fallback auf C62)
- IBAN/BIC im `SpecifiedTradeSettlementPaymentMeans` (Code 58 = SEPA)
- Buyer-Reference (Leitweg-ID falls B2G, sonst Kundennummer)
- Sicherer XML-Escape (alle 5 Entitäten)

### packages/integrations/email — Nodemailer + Mustache
- `sendEmail(input)` mit nodemailer-Transporter (SMTP-Config aus ENV)
  - Lokal: Mailpit auf Port 1025 (out-of-the-box, keine Auth)
  - Prod: einfach SMTP-Vars setzen oder Resend wrappen
- `renderTemplate(tpl, ctx)` für `{{Foo.bar}}` Mustache-Lite (nested object lookup)
- Attachments werden weitergereicht (PDF + optional XML)

### Documents-Router neue Procedures
- **`generatePdf`**: Rendert PDF, lädt zu Storage Bucket `documents` hoch (`{tenantId}/document/{id}/{number}.pdf`), persistiert `pdfStorageKey` in der DB-Zeile, gibt 1h-signed-URL zurück
- **`sendByEmail`**: Rendert PDF (+ optional XML), wendet Mustache-Templating auf Subject/Body an (mit Document/Contact/Company/User-Kontext), versendet via SMTP, schreibt in `email_outbox` mit `messageId`, setzt Document auf `status="sent"` + `sentAt`, schreibt Logbuch-Event

### Next.js Routes
- **`GET /api/documents/[id]/pdf`** — streamt das frisch gerenderte PDF (inline standardmäßig, `?download=1` für attachment)
- **`GET /api/documents/[id]/xml`** — streamt XRechnung-XML als Download
- Beide auf `runtime: "nodejs"` (React-PDF braucht Node), Auth via Auth.js Session, Tenant-Scope erzwungen

### UI: DocumentActions erweitert
- Drei neue Buttons: **PDF** (öffnet `/api/documents/[id]/pdf` in neuem Tab), **XRechnung** (lädt XML), **Versenden** (öffnet Dialog)
- Versenden-Button disabled wenn Kontakt keine E-Mail hat (mit Tooltip)
- **`SendEmailDialog`**: Modal mit To/CC/Subject/Body (Mustache-Hinweis), Checkbox „XRechnung-XML zusätzlich anhängen", Mailpit-Hinweis im Footer

## Kompletter End-to-End-Flow jetzt möglich
1. Login → Steiner-Projekt → Angebot AN-2026-001
2. **PDF** Button → professionelles A4-PDF mit allen 10 Positionen, USt-Breakdown, Bankdaten im Footer
3. **XRechnung** Button → standalone XML, validiert gegen CEN/EN 16931, geeignet für B2G-Versand in DE/AT
4. **Versenden** Button → Dialog → To vorbelegt mit Kontakt-E-Mail, Subject+Body als Mustache-Templates → Senden → PDF+XML-Anhang per SMTP an Mailpit (lokal) → Logbuch-Eintrag „Angebot AN-2026-001 an klaus.steiner@example.at versendet"
5. Document-Status springt auf `sent`, `sent_at` wird gesetzt
6. **Abschließen** → `locked=true`, GoBD-konform unveränderbar
