# Status 2026-04-22 (Update — Modul M12 FlowAI komplett mit Demo-Modus)

## Done in dieser Runde

### FlowAI — das Differenzierungs-Feature live (CLAUDE.md B.2)
Die UI-Schicht für **Modul M12 FlowAI** ist jetzt da. Das Backend in `packages/ai` stand schon, gemappt wird jetzt 1:1 durch einen neuen tRPC-Router + Pages.

**`flowai` tRPC-Router** (`packages/api/src/routers/flowai.ts`) — alle Procedures **module-gated** via `moduleProcedure(FEATURES.M12_FLOW_AI)`:
- `status()` — zeigt `{mode: "live" | "demo", model}` — UI kann Warn-Badge zeigen wenn Key fehlt
- `photoToOffer({image, mediaType})` — base64-Bild → Claude-Vision-Analyse aus `packages/ai/src/photo-to-offer.ts`
- `mailToProject({from, subject, body})` — strukturiertes JSON aus Mail-Inhalt
- `ocrReceipt({file, mediaType})` — PDF/Bild → Rechnungsdaten + Positionen
- `createFromMail({contact, project})` — One-Click-Aktion: erstellt Kontakt + Projekt aus Mail-Extraktion in einer Transaktion, mit Auto-Kundennummer, Auto-Projektnummer und Logbuch-Event
- **Demo-Modus**: Wenn `ANTHROPIC_API_KEY` fehlt ODER `demo: true` übergeben wird, liefert jede Procedure realistische Beispiel-Daten (800–1000ms simulated latency) mit `_demo: true` Flag — so funktioniert der End-to-End-Flow OHNE Key

### FlowAI-Seiten (`/flowai/*`)

**`/flowai`** Hub-Seite neu:
- 5 Feature-Kacheln (Foto→Angebot, Mail→Projekt, OCR-Beleg live; Sprache→Projekt, Assistenz-Panel disabled/bald)
- Live/Demo-Badge im Header basierend auf `status()`
- Prominente Demo-Mode-Info-Card wenn Key fehlt

**`/flowai/photo-to-offer`** — Flagship-Feature:
- Aspect-4/3 Drop-Zone mit Foto-Preview + ⨯-Clear
- „Analysieren"-Button triggert Claude Vision
- Ergebnis in 4 Cards: **Bestandsanlage** (Typ/Brand/Baujahr/kW + Konfidenz-Badge), **Empfohlene Wärmepumpen** (Top-3 mit Badge in primary/neutral), **Hydraulik-Anpassungen** (4 Check-Items mit grünem Häkchen), **Offene Klärungspunkte** (Bullet-Liste)
- Im Demo-Modus: realistische „Viessmann Vitorondens 200-T / Vitocal 350-G 10kW" Empfehlung

**`/flowai/mail-to-project`** — Split-View:
- Links: Textarea mit **Sample-Mail vorausgefüllt** (Max Mustermann Klagenfurt Neubau) — realistisches Beispiel
- Client-seitige Vorverarbeitung: Extrahiert `Von:`/`Betreff:`/Body aus Paste
- Rechts: Klassifizierung (intent/urgency/trade) + Kontakt-Preview + Projekt-Zusammenfassung + **„Kontakt + Projekt anlegen"-Button** → springt nach Create direkt auf das neue Projekt
- createFromMail-Aktion persistiert beide Entitäten atomar mit Logbuch „FlowAI Mail-Extraktion"

**`/flowai/ocr-receipt`** — Supplier-Invoice-Parser:
- Aspect-3/4 Drop-Zone für PDF oder Bild (16 MB Limit)
- Bild-Preview wird direkt gezeigt, PDF als Filename+Icon
- Ergebnis: Lieferant-Card (mit Match-Confidence-Badge), Rechnungsmeta (Nummer/Datum/Fälligkeit/Summen), Positions-Tabelle (Art.-Nr./Bezeichnung/Menge/EP/Gesamt), SKR03/04-Buchungs-Vorschläge, Projekt-Hinweise
- Im Demo: „Frauenthal-Rechnung FR-2026-4471 über €5.784 mit 4 Positionen → Projekt P-2026-001 Steiner"

1. Monteur macht Foto im Heizungsraum → Upload nach `/flowai/photo-to-offer`
2. Claude erkennt „Öl-Kessel Viessmann Vitorondens 200-T, ca. Bj. 2008, 24kW" → empfiehlt Vitocal 350-G 10kW + 800L-Puffer + R290-Alternative
3. Chef prüft Vorschlag → Button „Als Angebot" (kommt noch — Handoff zum Dokumenten-Editor)
4. Kunde antwortet per Mail → Copy/Paste in `/flowai/mail-to-project`
5. System extrahiert Kunde + Umfang + Dringlichkeit → **1 Klick** → Kontakt + Projekt + Logbuch angelegt
6. Beleg vom Großhandel kommt → Upload in `/flowai/ocr-receipt`
7. System erkennt Lieferant, liest Positionen, schlägt Projekt + DATEV-Konto vor
