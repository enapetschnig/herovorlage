## Done in dieser Runde

### Modul M14 — Projekt-Kanban + Chat
**`kanban` tRPC-Router** mit `board()` und `moveCard()`:
- Lädt alle aktiven Projekte (status NICHT in paid/cancelled) gruppiert nach Status mit Joined-Daten (Kontakt, Verantwortlicher, Potential)
- 7 Lanes: Lead → Angebot → Angenommen → Geplant → In Arbeit → Fertig → Fakturiert (mit eigenen Farb-Codes)
- `moveCard` schreibt Status-Wechsel + Logbuch-System-Event mit Lane-Label

**`projectMessages` tRPC-Router** mit `list / send / unreadCount`:
- Pro Projekt eigener Chat-Thread (DB: `project_messages` Tabelle)
- Auto-refetch alle 15s für Quasi-Realtime-Feel

**`/projects/kanban` Seite** mit **dnd-kit Drag&Drop**:
- 7 horizontal scrollbare Lanes, jede mit Farb-Dot, Karten-Anzahl, Wert-Summe
- Karten: Projekt-Nr, Title (klickbar), Kunde, Potential-Badge, verantwortlicher User
- **DragOverlay**: gezogene Karte folgt dem Cursor mit Schatten und Ring
- Optimistic Update der lokalen Lanes + tRPC-Persist im Hintergrund (rollback bei Fehler via `router.refresh()`)
- Drop-Zone visuell hervorgehoben (primary-Tint + Ring) während Drag
- Klick auf Karten-Title navigiert zum Projekt (stoppt Drag-Propagation)

**Chat-Tab auf Projekt-Detail** (`<ProjectChatTab>`):
- Slack-style Thread mit Avatar + Nachname + relativem Zeitstempel
- Auto-Scroll zum neuesten Eintrag
- Cmd+↩ zum Senden, Mehrzeilen-Textarea
- 15s Polling (in V1 ohne WebSocket — reicht für Team-Chat)

**Kanban-Button** in der Projekte-Toolbar zwischen „Neues Projekt" und Filter

### Modul M11 — SEPA-Lastschrift + Mahnwesen
**`packages/integrations/sepa`** — Vollständiger pain.008.001.08-Generator:
- SEPA-Direct-Debit (CORE) für B2C-Lastschriften
- Mandate-Daten pro Debitor: `mandateId`, `mandateDate`, `sequenceType` (FRST/RCUR/OOFF/FNAL)
- **`transliterate()`** — Umlaut-Mapping (ä→ae, ß→ss …) für SEPA-ASCII-Compliance
- **`isValidIban()`** — Mod-97-Checksumme + Format-Check (alle 4 Test-IBANs korrekt klassifiziert)
- Korrekte ISO-20022-Namespace und Pflicht-Pfade (CdtrSchmeId, ChrgBr=SLEV, SvcLvl/Cd=SEPA, LclInstrm/Cd=CORE)

**`reminders` tRPC-Router** mit:
- `markOverdue()` — setzt alle `status='sent'`-Rechnungen mit `dueDate < today` auf `overdue`, RETURNING-Liste der Nummern
- `overdueList()` — Liste aller überfälligen Rechnungen mit Joined-Kontakt, Tage-überfällig + E-Mail
- `sendReminder({documentId, level: 1|2|3})` — generiert Mahn-Mail mit:
  - **3 Mahnstufen** mit eigenen Subjects + Bodies + steigenden Mahngebühren (0/5/10 €)
  - PDF der Original-Rechnung re-generiert und angehängt
  - Mustache-Templating für Subject + Body mit `{{Document.number}}`, `{{Reminder.fee}}`, `{{Contact.lastName}}` etc.
  - Eintrag in `email_outbox` mit Message-ID
  - Logbuch-Event auf Kontakt
- `markPaid({documentId})` — setzt Status auf `paid`, Logbuch-Event

**`/reminders` Seite (Mahnwesen-UI)**:
- 4 KPIs: Überfällige Rechnungen / Offen gesamt / Über 14 Tage / Über 30 Tage (mit Farb-Eskalation)
- Tabelle mit Tagen-Überfällig-Badge (gelb >14, rot >30)
- **3 Mahn-Buttons pro Zeile** (M1 ghost / M2 secondary / M3 danger) mit Confirm-Dialog
- „Bezahlt"-Button für manuelle Markierung
- „Status neu prüfen"-Button löst `markOverdue` aus
- SEPA-Info-Card am Ende mit Hinweis auf das XML-Format

**Sidebar erweitert** um „Mahnwesen" mit AlertCircle-Icon

## App-Funktionalität jetzt End-to-End
1. ⌘K → „Kanban" → `/projects/kanban` → 7 Spalten mit allen aktiven Projekten → Steiner-Karte von „In Arbeit" auf „Fertig" ziehen → Status wird live aktualisiert + Logbuch-Eintrag „Status auf 'Fertig' geändert (Kanban)"
2. Alpenhotel-Projekt → Tab **Chat** → Nachricht „Pufferspeicher kommt morgen Vormittag" → erscheint sofort → Kollege auf einem anderen Browser sieht das nach max. 15s
3. Sidebar **Mahnwesen** → 4 KPI-Kacheln + Liste überfälliger Rechnungen → bei jeder Rechnung 3 Mahn-Stufen-Buttons → klick auf M2 → Confirm → Mail mit Original-PDF + Mahngebühr €5 wird per Mailpit versendet, Logbuch-Eintrag erscheint, Outbox füllt sich
