# Status 2026-04-22 (Update — Modul M3 komplett + CSV-Import)

## Done in dieser Runde

### Modul M3 — Wartung & Anlagen (CLAUDE.md Teil M.5)
Komplett von Backend bis UI — das ist das Differenzierungs-Feature für Wärmepumpen-Betriebe.

**`maintenance` tRPC-Router** (`packages/api/src/routers/maintenance.ts`) — alle Procedures sind **module-gated** via `moduleProcedure(FEATURES.M3_MAINTENANCE)`:
- `contractsList({ search?, dueWithinDays? })` — Verträge mit Kontakt + Anlage joined, sortiert nach Fälligkeit
- `contractById({ id })` — voller Vertrag + Kontakt + Anlage + Visit-Historie (alle jemals geplanten + abgeschlossenen Termine)
- `createContract(...)` — erzeugt Vertrag UND erste geplante Visit automatisch
- `upcomingVisits({ days })` — alle noch nicht abgeschlossenen Termine mit Contact + Technician joined
- `completeVisit({ id, protocol, issuesFound, followUpRequired })` — markiert Visit fertig, **rollt `next_due_date` um `interval_months` weiter UND erzeugt den Folgetermin automatisch**, schreibt Logbuch-Event
- `listAssetsByContact / createAsset / removeAsset`
- `dashboard()` — KPIs für Dashboard (Verträge, Überfällig, 30/60-Tage-Fenster, Anlagen-Total)

**Maintenance-Seiten:**
- `/maintenance` — KPI-Grid (5 Kacheln) + anstehende Wartungstermine (60 Tage) mit Ampel (rot=überfällig, gelb=<7 Tage, grün=OK) + Vertragstabelle
- `/maintenance/[id]` — 3-Spalten-Layout: Links **Anlagen-Details** (Marke, Modell, Seriennr., kW, COP, Kältemittel, Schall, Installations-/Garantiedatum, Standort); rechts oben **Nächster-Termin-Card** mit „Wartung abschließen"-Button; rechts Visit-Historie als Timeline (erledigte mit grünem Dot, offene mit grauem)
- `<CompleteVisitButton>` öffnet Modal mit **8-Punkt-Checkliste** aus CLAUDE.md Teil M.5 (Kältekreislauf, Verdampfer/Kondensator, Sole, Sicherheitsventile, Elektrik, Fehlerspeicher, Druckprüfung, Kunden-Unterschrift), Mängel-Feld, Folgeauftrag-Checkbox

**Anlagen-Tab auf Kontakt-Detail:**
- `<ContactAssetsCard>` neue Card auf jeder Kontakt-Detail-Seite
- Anzeige: Typ-Label, Marke/Modell, Leistung/COP/Kältemittel, Seriennr. (mono), Installationsdatum
- „+ Anlage"-Button öffnet `<AssetDialog>` mit 10 Feldern (Typ, Marke, Modell, Seriennr., Kältemittel-Dropdown, kW, COP, Installations-/Garantiedatum, Standort)
- Sofortiges Löschen via Trash-Icon pro Anlage

### CSV-Import für Kontakte
- **`contacts.importBulk(rows)`** Procedure: validiert jede Zeile einzeln (Companyname/First+Last-Name-Checks), generiert `customer_number` pro Zeile, läuft in Transaktion pro Row, gibt `{imported, failed, successes, errors}` zurück — **ein schlechter Datensatz blockt nicht den Rest**
- **`/contacts/import`** Seite mit 5-Schritte-Flow:
  1. File-Drop — **Tiny-CSV-Parser** inline (quoted fields, `\r\n`, trennt auf `,` / `;` / Tab)
  2. Defaults (Typ Kunde/Lieferant, Person/Firma)
  3. **Column-Mapping mit Auto-Erkennung**: erkennt deutsche+englische Header (Firma, Vorname, Nachname, Email, Telefon, Straße, PLZ, …) via Alias-Liste, pro Feld kann manuell überschrieben werden
  4. Tabellen-Vorschau (erste 10 Zeilen mit gemappten Spalten)
  5. Import mit Progress-Anzeige + Ergebnis-Report inkl. Fehler pro Row (expandable `<details>`)
- Button „CSV importieren" in der Kontakt-Liste-Toolbar
